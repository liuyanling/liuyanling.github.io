搭建静态博客：hexo+github

Veröffentlicht am 2016-03-10

hexo 是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github和Heroku上。
为什么要用hexo： 快速，支持Markdown，高扩展性，兼容Windows,Mac&Linus，易用，轻，用户多。
在搭建hexo博客的时候，用到的一些常用命令：
hexo new”postName” #新建文章

hexo new page”pageName” #新建页面

hexo generate #生成静态页面至public目录

hexo server #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）

hexo deploy #将.deploy目录部署到GitHub

hexo help #查看帮助

hexo version #查看Hexo的版本

总结：
第一次安装，磕磕碰碰的，出现各种问题，还好差不多都能从教程的评论里找到解决办法,一定要注意_config.yml里面的空格以及在何处打开git bash，是站点配置文件还是主题配置文件要看清楚。除了选择主题的时候，每次在本地时，主题应用上了，可是部署到github pages上就各种css加载不出来，表示头疼得不要不要的，把网上能找的教程差不多翻遍了，硬是没找到答案。后来在加入的hexo群里问了下，有人说是主题的设置不对，就又开始看关于主题Next的教程，能改动的都改动了，发现并没有什么用。没办法，请教了一个大牛同学，人家一眼就看出来了原因： IE8不支持该主题，换个浏览器试试。试了试，还真是这个原因。一度泪眼汪汪了，换个浏览器！！！之前各种改改修修删删的，根本就没找到问题所在，浪费我多少时间。哎，技术党就是不一样！！！当然，我这样的技术小白居然也搭建了一个静态博客，虽然坎坷，可是通过看博客，看教程，看官网，总算是成功了。
